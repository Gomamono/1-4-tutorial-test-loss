## 使用方法

2. チュートリアル中の「[3] 損失関数とオプティマイザの定義」まで実行する．
3. 貼り付けたコードを実行する．

# AIによるREADME

## 📌 なぜこの検証コードを作成したのか？（背景と目的）

  PyTorchの公式チュートリアル（CIFAR-10を用いた画像分類）を学習する際、初学者が陥りやすい**「ある誤解」**を解き明かすために本コードを作成しました。

  ### ⚠️ 初学者が陥りやすい誤解

  公式のチュートリアルノートブックを上から順に実行すると、以下のような現象が観察されます。

  • CPUでの学習開始時: Lossが  2.3  付近からスタートする
  • GPUでの学習開始時: Lossが  1.2  付近からスタートし、最終的により低い値になる

  この出力結果だけを見ると、**「GPUで学習した方がAIが最初から賢く、より良い精度（Loss）が出るのではないか？」**というデバイス間の有意差を錯覚してしまいます。

  ### 🔍 錯覚が起きる原因

  この「有意差があるように見える現象」は、以下の要因によるものです。

  1. 学習済み重みの引き継ぎ: チュートリアルのGPUコード部分は、モデル（ net
  ）を初期化せずに、CPUですでに2エポック学習した状態の続き（実質3〜4エポック目）を実行しています。
  2. 乱数シードの非固定: 重みの初期値やデータのシャッフル順が実行のたびにランダムになるため、運の要素でLossの下がり方にバラつきが生じます。
  3. 本来の目的: 公式チュートリアルは「CPUとGPUの精度比較」ではなく、「 .to(device)
  というGPU転送の書き方を教えること」を目的としているため、厳密な比較実験の構成になっていません。

  ### 💡 本コードの目的とアプローチ

  本検証コードは、この誤解を解き、**「CPUとGPUで行われている数学的な学習プロセスは（実用上）完全に同一である」**ことを証明するために書かれました。

  そのために、本コードでは公式チュートリアルに以下の厳密な比較条件を追加しています。

  •  set_seed()  による乱数の完全固定: CPU・GPUともに全く同じランダム初期値・同じデータ順序からスタートさせます。
  • ネットワークの毎回の初期化: 比較のたびに  test_net = Net().to(device)  と呼び出し、学習状態をリセットして同じスタートラインに立たせます。

  このコードを実行し可視化されたグラフを見ることで、計算アーキテクチャの違いによるミクロな丸め誤差（0.
  002程度）は生じるものの、学習の軌跡（Lossの推移）には有意差がないことを明確に確認することができます。
